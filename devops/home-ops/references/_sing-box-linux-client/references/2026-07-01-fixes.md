# 2026-07-01: Script Fixes + Proxy Toggle + LAN Access

## Bugs Fixed in sing-box-ctrl.py

### `cmd_test` — Latency measurement (major)

Root cause: `curl_probe()` returned `r.stderr`, but `-w "%{time_connect}"` writes to stdout.

Before (broken):
```python
def curl_probe(*args, timeout=30) -> tuple[int, str]:
    r = subprocess.run(...)
    return r.returncode, r.stderr.strip()

# caller:
rc, _ = curl_probe(...)
if rc == 0:
    lats.append(0)            # ← hardcoded 0
```

After (fixed):
```python
def curl_probe(*args, timeout=30) -> tuple[int, str]:
    """返回 (returncode, stdout)"""
    r = subprocess.run(...)
    return r.returncode, r.stdout.strip()

# caller:
rc, out = curl_probe(...)
if rc == 0 and out.strip():
    lats.append(float(out.strip()) * 1000)   # ← real timing data
```

### `cmd_test --all` — Node switching (major)

Root cause: Temp config has flat outbounds (no selector), but switching code looked for a selector outbound to change `default`.

Before: `selector[0]["default"] = tag` — selector doesn't exist → no-op → all tests run on same node.

After: `test_cfg["route"]["final"] = tag` — changes `route.final` directly (flat config pattern).

### `cmd_start` Windows — Missing call parentheses

`PLAT.is_running` → `PLAT.is_running()` (was printing method repr).

### Dead code removed

- `plugin_import()` — never called
- `PROXY_ENV_FILE` — defined but unused (code used inline `Path.home() / ".config/proxy-env"`)
- `test_urls` list — defined but never used (actual URLs hardcoded in curl calls)

## New Feature: `proxy` Subcommand

See "System Proxy Toggle" in SKILL.md. Key implementation details:

### GUI (gsettings)
```python
gsettings set org.gnome.system.proxy mode 'manual'
gsettings set org.gnome.system.proxy.http host '127.0.0.1'
gsettings set org.gnome.system.proxy.http port 10881
gsettings set org.gnome.system.proxy.socks host '127.0.0.1'
gsettings set org.gnome.system.proxy.socks port 10880
```

### CLI (env file sourcing)
- Writes `~/.config/proxy-env` with export/unset lines
- `~/.bashrc` sources: `[ -f "$HOME/.config/proxy-env" ] && . "$HOME/.config/proxy-env"`

### Limitation
Env vars can't propagate from child (Python) to parent (shell). Script always prompts:
```
  ⚠ 当前终端环境变量未更新
  请运行:
    source ~/.config/proxy-env
```

## Flat Config Switching Pattern

Never assume a `selector` outbound exists. The primary switching mechanism is `route.final`:

```python
route = cfg.setdefault("route", {})
route["final"] = new_node
save_config(cfg)
PLAT.reload(pid)
```

`current_node()` reads `route.final` first, falls back to selector:

```python
def current_node(self, cfg: dict) -> str:
    route = cfg.get("route", {})
    final = route.get("final", "")
    if final:
        return final
    selector = [o for o in cfg.get("outbounds", []) if o.get("type") == "selector"]
    if selector:
        return selector[0].get("default", "") or selector[0].get("outbounds", [""])[0]
    return "—"
```

## TUN Mode: Final Verdict

After 4 attempts all causing network outages, **TUN mode abandoned** on this Linux Mint system. The `tun on/off/status` code was removed from `sing-box-ctrl.py`. The rollback script (`sing-box-tun-rollback.sh`) deleted, `cap_net_admin` removed from binary.

Root causes of failures:
1. `dns_mode: "hijack"` (sing-box 1.14+) + `sniff` (removed in 1.13) → crash → nftables rules remain → total outage
2. `strict_route: false` → no fwmark bypass → node connection loop → international traffic dead
3. `strict_route: true` + `route_exclude_address_set` → potential nftables conflict with NetworkManager → total outage
4. User's own retry after our config change → outage

## LAN Access (0.0.0.0 Binding)

Changed SOCKS5 and Mixed inbound listen from `127.0.0.1` to `0.0.0.0` to allow other devices on the LAN to use this machine as a proxy.

**Config change**: `"listen": "127.0.0.1"` → `"listen": "0.0.0.0"` for both `socks-in` and `mixed-in` inbounds.

**Firewall**: ufw must allow traffic from LAN subnets:
```bash
sudo ufw allow from 192.168.0.0/16 to any port 10880 proto tcp
sudo ufw allow from 192.168.0.0/16 to any port 10881 proto tcp
```

**Verification from another LAN device**:
```bash
curl -x socks5://192.168.71.24:10880 https://www.google.com
```

## OpenClash Integration (ImmortalWrt)

Add the local sing-box SOCKS5 as a node on OpenClash so LAN traffic can route through it:

```yaml
# In /etc/openclash/config.yaml (and /etc/openclash/config/config.yaml)
- name: lenovo-socks
  type: socks5
  server: 192.168.71.24        # This machine's LAN IP
  port: 10880
```

Then add `lenovo-socks` to the PROXY group's proxy list:
```yaml
proxy-groups:
- name: PROXY
  type: select
  proxies:
  - Alibaba-Seoul-VLESS-Reality
  - 233boy-KVM
  - Seoul-Cloudflare
  - VMISS-HK
  - AUTO
  - lenovo-socks
```

Both `/etc/openclash/config.yaml` and `/etc/openclash/config/config.yaml` must be edited (OpenClash reads from both).

Verify via API:
```bash
curl -s -H "Authorization: Bearer <secret>" \
  http://192.168.71.9:9090/proxies | jq '.proxies.PROXY'
```
