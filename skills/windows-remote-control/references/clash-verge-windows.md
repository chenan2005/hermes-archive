# Clash Verge Rev on Windows — WiFi-Bound Proxy for Router

## Use Case

Run Clash Verge on a Windows machine (minipc) that has both Ethernet (LAN, 71.21) and WiFi (phone hotspot). The proxy connections (VLESS/etc.) should go through WiFi for 5G speed, while the SOCKS5 server listens on the LAN for the router (ImmortalWrt) to consume.

## Key Configuration Files

All located under `%APPDATA%\io.github.clash-verge-rev.clash-verge-rev\`:

| File | Purpose | Editable? |
|------|---------|-----------|
| `clash-verge.yaml` | **Runtime config** — Generated by Clash Verge from active profile + app settings. Contains nodes, groups, rules, and global settings like `interface-name`, `allow-lan`. | ✅ Can be patched while Clash Verge is stopped. **Will be regenerated on next startup.** |
| `config.yaml` | Another generated config. Gets overwritten on startup. | ⚠️ Not persistent. |
| `profiles.yaml` | Profile index — lists available config profiles, which one is active. | ✅ Editable. |
| `profiles/*.yaml` | Individual profile config files (user-created profiles). | ✅ These persist. |
| `verge.yaml` | Clash Verge **app settings** — language, theme, ports, tray settings. | ✅ Editable. |

### Key Finding: `clash-verge.yaml` Is the Runtime Config

When Clash Verge starts, it reads `profiles.yaml` → loads the active profile → merges with app overlay settings → writes `clash-verge.yaml`. The mihomo core then loads `clash-verge.yaml`.

**This means:** Patching `clash-verge.yaml` while running works temporarily, but changes will be lost on restart. To make persistent changes:
1. Modify the profile file in `profiles/*.yaml`
2. Then the override settings through Clash Verge's UI

Alternatively, patch `clash-verge.yaml` while the process is stopped, then start Clash Verge.

## Critical Settings for Router Proxy

### `interface-name: WLAN`

Forces all mihomo outbound connections through the WiFi interface. Set this in the `clash-verge.yaml` (or profile) under global settings:

```yaml
interface-name: WLAN
```

**Verification:** After traffic flows, run on Windows:
```cmd
netstat -n -o | findstr <proxy-server-ip>
```
The source IP should be the WiFi IP (e.g., 192.168.1.x), not the LAN IP (192.168.71.x).

### `allow-lan: true`

Without this, the SOCKS5 listener binds only to 127.0.0.1 (localhost). Set to `true` to bind to 0.0.0.0 so the router can connect:

```yaml
allow-lan: true
```

**Before (`false`):**
```
TCP    127.0.0.1:7897         0.0.0.0:0              LISTENING
```

**After (`true`):**
```
TCP    0.0.0.0:7897           0.0.0.0:0              LISTENING
```

### Minimal Profile Example

The user explicitly wanted: "四个节点可以手动选择切换，然后绑定 WLAN" — no routing rules, no GeoIP, no DNS, just raw nodes:

```yaml
mixed-port: 7897
allow-lan: true
mode: global
interface-name: WLAN

proxies:
- name: "VMISS-HK"
  type: vmess
  server: vmiss.bernarty.xyz
  port: 443
  uuid: ac6aa939-156c-452f-a7da-4ddd79b7d5c9
  alterId: 0
  cipher: auto
  tls: true
  servername: vmiss.bernarty.xyz
  network: ws
  ws-opts:
    path: "/ws-vmiss"
    headers:
      Host: vmiss.bernarty.xyz

# ... add other nodes similarly ...

proxy-groups:
- name: PROXY
  type: select
  proxies:
  - VMISS-HK
  - 233boy-KVM
  - Seoul-Cloudflare
  - Alibaba-Seoul-VLESS-Reality
  - AUTO

- name: AUTO
  type: url-test
  url: https://cp.cloudflare.com/generate_204
  interval: 300
  tolerance: 100
  proxies:
  - VMISS-HK
  - 233boy-KVM
  - Seoul-Cloudflare
  - Alibaba-Seoul-VLESS-Reality
```

Notice: no `rules:` section, no `dns:` block. `mode: global` sends all traffic through the selected proxy.

## File Transfer to Windows Over SSH (No SCP)

When SCP is blocked or unavailable, use the pipe-through-PowerShell method:

```bash
# Write a config file from stdin to Windows
cat local_config.yaml | ssh target 'powershell -NoProfile -Command \
  "$i=[Console]::In.ReadToEnd(); \
   [IO.File]::WriteAllText(\"$env:APPDATA\\path\\to\\file.yaml\",\"$i\"); \
   echo ok"'
```

**Pitfall — encoding:** `Set-Content` defaults to UTF-16. Use `[IO.File]::WriteAllText()` for reliable UTF-8 output.

**Pitfall — process overwrites:** Clash Verge rewrites `clash-verge.yaml` and `config.yaml` on startup. Write files while the process is stopped, or write profile files (which persist).

## Removing Clash Verge and Cleanup

If the Windows proxy approach is abandoned:
1. Kill all Clash processes: `taskkill /F /IM "clash-verge.exe" /IM "verge-mihomo.exe"`
2. Delete app data: `rm -rf "$env:APPDATA\io.github.clash-verge-rev.clash-verge-rev\*"`
3. Uninstall: via Settings → Apps → Clash Verge Rev
4. Restore WiFi metric if changed
