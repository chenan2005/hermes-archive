# Windows 桌面环境 VRAM 开销

Windows 在 GPU 上分配的显存不全是给 LLM 推理用的。桌面合成、多显示器、浏览器 GPU 加速都会占用 VRAM。在估算本地模型能否跑得下时，这些开销必须计入。

## 桌面合成（DWM）

DWM（Desktop Window Manager）管理所有窗口的合成渲染：

| 配置 | VRAM 占用 | 说明 |
|------|----------|------|
| 单 1080p | ~300-500 MB | 基本桌面 + 任务栏 |
| 单 4K | ~500-800 MB | 高分屏需要更大的帧缓冲 |
| 双 4K | ~1-1.5 GB | 双倍合成 buffer + 跨屏缩放 |
| 三 4K | ~1.5-2.5 GB | 极限情况 |

## 浏览器 GPU 加速

| 状态 | VRAM 额外占用 |
|------|--------------|
| 浏览器关闭 | 0 |
| 浏览器打开，静态页面 | ~200-300 MB |
| 浏览器 + YouTube 4K | +700 MB - 1.2 GB（解码帧缓冲 + 合成） |
| 浏览器 + 多个视频/动效标签 | +1.5-2 GB |

YouTube 4K 视频通过 GPU 的 NVDEC 硬件解码器解码（不占 CUDA 核心），但解码后的帧缓冲存放在 VRAM 中。这是额外的 ~200-400 MB 连续分配。

## IDE / 开发工具

| 工具 | VRAM 额外占用 |
|------|--------------|
| VS Code / Cursor | ~100-300 MB（Electron GPU 合成） |
| Visual Studio | ~200-500 MB |
| JetBrains IDE | ~200-400 MB |
| Unity Editor | ~500 MB - 2 GB（看场景复杂度和渲染管线） |

## 编译器 GPU 占用

大多数编译（MSVC, Clang, Rustc）是纯 CPU 任务，不占 GPU VRAM。但如果 IDE 同时在后台渲染（编辑器的语法高亮动画、滚动合成），会有持续的微量占用。

## NVIDIA 驱动常驻

| 组件 | VRAM |
|------|------|
| NVIDIA 控制面板/驱动 | ~100-200 MB |
| CUDA context（如果加载了） | ~500 MB - 1 GB |
| CUDA graph capture pools | ~500 MB - 1 GB |

## 合计参考（代码分析场景）

你的典型配置应该是：双 4K 显示器 + IDE 打开 + 浏览器几个标签（可能看文档）+ 模型推理在后台。

| 场景 | VRAM 开销 |
|------|----------|
| 轻量（单 4K + IDE） | ~1-1.5 GB |
| 中等（双 4K + IDE + 浏览器文档） | ~2-3 GB |
| 重度（双 4K + IDE + 浏览器 + YouTube 4K） | ~3-4 GB |

## 推荐估算方式

在做 VRAM budget 时，直接给 **3 GB** 作为系统开销基准值（双 4K + IDE + 浏览器）。如果用户说同时在看视频，加到 3.5-4 GB。这已经包含了 DWM + 浏览器 + IDE + NVIDIA 驱动的常规开销，不需要再逐项加。
